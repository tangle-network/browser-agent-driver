# Design Audit Guide

`bad design-audit` is a vision-powered design quality analysis system. It loads pages in a real browser, captures screenshots, extracts design tokens from the DOM, and uses an LLM with vision to produce structured findings with concrete CSS fixes.

## Quick Start

```bash
# Audit a live site
bad design-audit --url https://your-app.com --profile general

# Audit a local dev server
bad design-audit --url http://localhost:3000 --profile vibecoded

# Audit with JSON output
bad design-audit --url https://your-app.com --profile marketing --json
```

## Profiles

Profiles inject domain-specific evaluation rubrics. Choose the one that matches your app.

| Profile | Target | Calibration Reference | Score Range |
|---------|--------|----------------------|-------------|
| `general` | Any website | Linear/Stripe = 9, templates = 5 | 1-10 |
| `saas` | SaaS applications | Linear/Notion = 9, admin templates = 5 | 1-10 |
| `marketing` | Landing pages | Stripe/Vercel = 9, startup pages = 5 | 1-10 |
| `defi` | DeFi/crypto apps | Uniswap/Aave = 8, average DEX = 5 | 1-10 |
| `vibecoded` | AI-generated / template apps | Cursor/v0 = 9, unmodified shadcn = 4 | 1-10 |

### The `vibecoded` Profile

Built specifically for apps generated by AI coding tools (Cursor, v0, bolt, etc.). It detects:

- **Template defaults**: unmodified shadcn/ui, MUI, Ant Design, Chakra. Score ceiling: 4 if detected.
- **AI slop patterns**: everything same visual weight, "centered column of cards" layout, no hierarchy.
- **Design system coherence**: border-radius count, color palette size, spacing grid adherence.
- **Craft signals**: custom icons vs Lucide dump, micro-interactions, designed empty/error states.
- **Agentic app specifics**: agent status indicators, streaming UI, tool call visualization, error recovery.

## CLI Reference

```
bad design-audit [options]
```

### Required

| Flag | Description |
|------|-------------|
| `--url <url>` | URL to audit (local or remote) |

### Common Options

| Flag | Default | Description |
|------|---------|-------------|
| `--profile <name>` | `general` | Audit profile: general, saas, marketing, defi, vibecoded |
| `--pages <n>` | `5` | Max pages to discover and audit via link crawling |
| `--model <model>` | `sonnet` | LLM model for vision analysis |
| `--provider <name>` | `claude-code` | LLM provider: claude-code, openai, anthropic |
| `--json` | false | Also output machine-readable `report.json` |
| `--headless` | true | Run browser headlessly |
| `--viewport <WxH>` | `1440x900` | Viewport size |
| `--sink <dir>` | auto | Output directory for reports and screenshots |
| `--debug` | false | Print raw LLM responses |

### Modes

| Flag | Description |
|------|-------------|
| `--extract-tokens` | Pure DOM extraction (no LLM). Outputs colors, typography, spacing, components. |
| `--design-compare --compare-url <url>` | Pixel + token diff between two URLs across viewports. |
| `--rip` | Download full site assets (HTML, CSS, JS, fonts, images). |
| `--evolve <mode>` | Closed-loop fix: `css` (inject into browser), `claude-code`, `codex`, `opencode`, or custom command. |
| `--evolve-rounds <n>` | Max fix-reaudit cycles (default: 3). |
| `--project-dir <path>` | Source directory for agent to edit (defaults to cwd). |
| `--reproducibility` | Run 3x, report score variance. Target: +/-0.5 stddev. |

## Output

Every audit produces a directory with:

```
audit-results/<hostname>-<timestamp>/
  report.md           # Human-readable report with scores, findings, CSS fixes
  report.json         # Machine-readable (with --json)
  screenshots/
    index.png         # Homepage screenshot
    about.png         # Per-page screenshots
```

### Report Structure

Each page gets:

1. **Score** (1-10) with calibration bar
2. **Summary** — one-sentence assessment
3. **Strengths** — evidence-based positives
4. **Design System Breakdown** — 8 dimensions scored individually:
   - Layout, Typography, Color, Spacing, Components, Interactions, Accessibility, Polish
5. **Findings table** — each with:
   - Severity (critical/major/minor)
   - Category (layout/typography/spacing/contrast/etc.)
   - Description with measured values
   - Location with CSS selector
   - Suggestion with concrete CSS fix

## Design Token Extraction

Extract the full design system from any URL without an LLM call:

```bash
bad design-audit --url https://your-app.com --extract-tokens
```

Outputs a `tokens.json` with:
- CSS custom properties (all `--var` definitions)
- Color palette with semantic clustering (primary, secondary, accent, neutral)
- Typography: font families, weights, type scale with usage counts
- Spacing values with grid unit detection
- Border radius, shadows, animations
- Component fingerprints (buttons, inputs, cards, nav)
- Brand metadata (favicon, OG image, theme color)
- Detected libraries (shadcn, MUI, Tailwind, GSAP, etc.)
- All of the above at mobile (375px), tablet (768px), and desktop (1440px)

## Design Compare

Side-by-side comparison of two URLs:

```bash
bad design-audit --url https://v1.your-app.com \
  --design-compare --compare-url https://v2.your-app.com
```

Produces per-viewport:
- Pixel diff percentage
- Added/removed colors, fonts, CSS variables
- Component count deltas (buttons, inputs, cards)
- Brand metadata changes

## Site Rip

Download a full site for local analysis:

```bash
bad design-audit --url https://reference-site.com --rip --pages 10
```

Downloads HTML, CSS, JS, fonts, images with rewritten local paths.

## The Evolve Loop

`--evolve` has two modes:

1. **CSS injection** (default) — injects fixes into the browser page. Fast, ephemeral. Good for testing what CSS changes would help.
2. **Agent dispatch** (`--evolve claude-code|codex|opencode`) — sends findings to a real coding agent that edits your actual source files. Permanent, real fixes.

### Mode 1: CSS Injection

```bash
bad design-audit --url http://localhost:3000 --profile vibecoded --evolve css
```

```
Round 0: Initial audit
  -> Score: 4.5/10, 18 findings

Round 1: Extract CSS fixes from findings
  -> 12 findings have cssSelector + cssFix
  -> Inject fixes via page.addStyleTag()
  -> Re-audit with fixes applied
  -> Score: 5.8/10 (+1.3)

Round 2: Generate new fixes for remaining issues
  -> LLM generates 8 more targeted CSS fixes
  -> Inject cumulative fixes
  -> Re-audit
  -> Score: 6.5/10 (+0.7)

Round 3: Converged (delta < 0.1)
  -> Final: 4.5 -> 6.5 (+2.0)
  -> Outputs: design-fixes.css, evolve-report.md
```

### Usage

```bash
# Basic evolve (3 rounds)
bad design-audit --url http://localhost:3000 --profile vibecoded --evolve css

# More rounds for stubborn issues
bad design-audit --url http://localhost:3000 --profile vibecoded --evolve css --evolve-rounds 5

# With JSON tracking
bad design-audit --url http://localhost:3000 --profile vibecoded --evolve css --json
```

### Output

```
audit-results/<hostname>-<timestamp>/
  report.md                  # Initial audit report
  report.json                # Initial audit data (with --json)
  evolve-report.md           # Evolve loop summary with score progression
  evolve.json                # Machine-readable evolve data (with --json)
  design-fixes.css           # Generated CSS override file
  screenshots/               # Initial screenshots
  screenshots-round-1/       # Post-fix screenshots per round
  screenshots-round-2/
```

### Applying the Fixes

The evolve loop produces a `design-fixes.css` file. Apply it to your app:

```html
<!-- Drop into your HTML -->
<link rel="stylesheet" href="design-fixes.css">
```

```tsx
// Or import in your app
import './design-fixes.css'
```

```css
/* Example generated fixes */

/* major: Hero section has 64px top padding but only 16px bottom — inconsistent rhythm */
main > section:first-child { padding-bottom: 48px }

/* major: Body text #6b7280 on #f9fafb card fails WCAG AA (3.8:1) */
.card p, .card span { color: #4b5563 }

/* minor: 6 distinct border-radius values — consolidate to 3 */
.btn { border-radius: 8px }
.card { border-radius: 12px }
.avatar { border-radius: 9999px }
```

### Convergence

The evolve loop stops when:
1. Max rounds reached (default: 3)
2. Score improvement < 0.1 between rounds (converged)
3. No more major/critical findings remain

### CSS Injection Limitations

- Ephemeral — fixes disappear on page reload
- Can't fix HTML structure, component architecture, or JS behavior
- Works best for spacing, color, typography, and layout fixes
- Some findings require code changes (semantic HTML, ARIA labels, loading states)

### Mode 2: Agent Dispatch

Sends the audit findings to a coding agent that edits your actual source files.

```bash
# Claude Code
bad design-audit --url http://localhost:3000 \
  --profile vibecoded \
  --evolve claude-code \
  --project-dir ~/my-app

# Codex
bad design-audit --url http://localhost:3000 \
  --profile vibecoded \
  --evolve codex

# OpenCode
bad design-audit --url http://localhost:3000 \
  --profile vibecoded \
  --evolve opencode

# Any custom CLI agent
bad design-audit --url http://localhost:3000 \
  --profile vibecoded \
  --evolve "aider --message"
```

`--project-dir` defaults to the current working directory, so you can skip it if you run from the project root.

**How it works:**

```
Round 0: Initial audit → Score: 4.2/10, 23 findings

Round 1: Dispatch to claude-code
  -> Sends prompt with all findings, CSS selectors, suggested fixes
  -> Claude Code reads the project, identifies Tailwind/CSS approach
  -> Edits actual source files (components, globals.css, tailwind.config)
  -> Waits 5s for hot reload
  -> Re-audit → Score: 6.1/10 (+1.9)

Round 2: Dispatch remaining findings
  -> Claude Code fixes typography scale and component consistency
  -> Re-audit → Score: 7.4/10 (+1.3)

Round 3: Converged (delta < 0.1)
  -> Final: 4.2 → 7.4 (+3.2)
```

**What the agent receives:**
- Current scores (overall + 8-dimension breakdown)
- All findings sorted by severity with:
  - Description with measured values
  - CSS selectors pointing to the problem
  - Concrete CSS fix suggestions
- Instructions to match the project's styling approach
- Instructions to fix the design system, not individual instances

**Built-in agents:**

| Agent | Command Used | Notes |
|-------|-------------|-------|
| `claude-code` | `claude -p "..." --dangerously-skip-permissions` | Full file access, reads project context |
| `codex` | `codex exec "..."` | Sandboxed execution |
| `opencode` | `opencode run "..."` | TUI-based agent |
| Custom | Your command + prompt as last arg | Any CLI that takes a prompt |

**Requirements:**
- `--project-dir` defaults to cwd — set it if running from a different directory
- The dev server must be running with hot reload (Vite, Next.js dev, etc.)
- The agent CLI must be installed and authenticated
- 5-minute timeout per agent round

**Output:**

```
audit-results/<hostname>-<timestamp>/
  report.md                        # Initial audit report
  evolve-report.md                 # Score progression and summary
  agent-prompt-round-1.txt         # Exact prompt sent to agent
  agent-output-round-1.txt         # Agent's stdout
  findings-round-1.json            # Structured findings for that round
  screenshots-round-1/             # Post-fix screenshots
```

**Debug mode** — see exactly what's being sent and what the agent does:

```bash
bad design-audit --url http://localhost:3000 \
  --profile vibecoded \
  --evolve claude-code \
  --project-dir ~/my-app \
  --debug
```

This pipes the agent's stdout/stderr to your terminal so you can watch it work.

## Reproducibility Testing

Verify score consistency:

```bash
bad design-audit --url https://stripe.com --profile marketing --reproducibility
```

Runs 3 audits of the same page and reports:
- Individual scores per run
- Mean score
- Standard deviation
- Pass/fail at +/-0.5 threshold

## Starter-Foundry Integration

starter-foundry generates project scaffolds — it has no UI itself. To audit starter-foundry outputs:

### 1. Generate a Project

```bash
cd ~/webb/starter-foundry
npm run build

# Plan a scaffold
npm run plan -- --prompt "Next.js SaaS dashboard with auth"

# Compose it
npm run compose -- --spec <spec.json> --out /tmp/test-app
```

### 2. Start the Generated App

```bash
cd /tmp/test-app
npm install
npm run dev
# Note the local URL (typically http://localhost:3000 or :4175)
```

### 3. Audit It

```bash
# Quick single-page audit
bad design-audit --url http://localhost:3000 --profile vibecoded

# Full multi-page crawl
bad design-audit --url http://localhost:3000 --profile vibecoded --pages 10

# Extract design tokens (no LLM cost)
bad design-audit --url http://localhost:3000 --extract-tokens

# Evolve loop — generate fixes
bad design-audit --url http://localhost:3000 --profile vibecoded --evolve css --json

# Compare two scaffold outputs
bad design-audit --url http://localhost:3000 \
  --design-compare --compare-url http://localhost:3001
```

### 4. Agentic Improvement Loop

For maximum impact, combine evolve with the generated CSS:

```bash
# Option A: Agent dispatch (edits source directly)
bad design-audit --url http://localhost:3000 \
  --profile vibecoded --evolve claude-code --evolve-rounds 3

# Option B: CSS injection (generates a CSS file to apply manually)
bad design-audit --url http://localhost:3000 \
  --profile vibecoded --evolve css --evolve-rounds 5 --json

# If using CSS injection, copy the generated CSS into the project
cp audit-results/localhost-*/design-fixes.css /tmp/test-app/src/

# Import it in the app's layout
echo 'import "./design-fixes.css"' >> /tmp/test-app/src/app/layout.tsx

# 4. Re-audit to verify
bad design-audit --url http://localhost:3000 --profile vibecoded --json

# 5. Compare before/after
bad design-audit --url http://localhost:3000 \
  --design-compare --compare-url http://localhost:3000
```

### Batch Auditing Multiple Scaffolds

To audit all starter-foundry families:

```bash
#!/bin/bash
# audit-all-scaffolds.sh

FAMILIES=("nextjs-ts" "react-vite" "astro" "sveltekit")
PORTS=(3000 3001 3002 3003)

for i in "${!FAMILIES[@]}"; do
  family="${FAMILIES[$i]}"
  port="${PORTS[$i]}"
  
  echo "=== Scaffolding $family ==="
  npm run compose -- --spec "{\"family\":\"$family\"}" --out "/tmp/scaffold-$family"
  
  cd "/tmp/scaffold-$family"
  npm install && npm run dev -- --port $port &
  sleep 5
  
  bad design-audit \
    --url "http://localhost:$port" \
    --profile vibecoded \
    --pages 5 \
    --evolve claude-code \
    --json \
    --sink "./audit-results/scaffold-$family"
  
  kill %1
  cd -
done
```

## Cost

Default provider is `claude-code` — uses your Claude Code subscription (no per-call API cost).
With `--provider openai` or `--provider anthropic`, each page audit costs one LLM vision call (~5-10k tokens).

| Operation | Pages | LLM Calls | Cost (claude-code) | Cost (openai gpt-5.4) |
|-----------|-------|-----------|--------------------|-----------------------|
| Single page audit | 1 | 1 | subscription | ~$0.15 |
| 5-page crawl | 5 | 5 | subscription | ~$0.75 |
| Evolve (3 rounds, 1 page) | 1 | 4-7 | subscription | ~$1.00 |
| Reproducibility (3 runs, 1 page) | 1 | 3 | subscription | ~$0.45 |
| Token extraction | 1 | 0 | free | free |
| Design compare | 2 | 0 | free | free |
| Site rip | N | 0 | free | free |
