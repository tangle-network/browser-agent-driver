## Thesis

Gen 6/7/7.1/7.2 made `bad` fast and architecturally sound on local fixture tasks. The competitive bench at v0.20.0 already showed bad beats browser-use on synthetic tasks. **But every task we've benched is a fixture we control.** That's not enough evidence to stake a claim on. Gen 8 ships proof against 10 hard REAL public-web flows with multi-rep validation, video evidence, deterministic external oracles, and a public-friendly artifact dashboard — all under the same rigor protocol that caught the Gen 7 single-run overclaim.

**This generation is mostly validation infrastructure, not agent runtime changes.** The agent code is mature enough; the question is whether it works on real things, and we don't have honest evidence.

## System Audit

### What exists and works (relevant to Gen 8)
- `pnpm bench:compete` runs framework × task × rep grids with bootstrap CIs, Wilson, Cohen's d, MWU
- 4 local fixture tasks: form-fill-multi-step, dashboard-extract, dashboard-edit-export, plus `local-long-form` from the scenario suite
- Per-run video recording (`recording.webm`) is ALREADY being generated by the test harness — just not surfaced in bench:compete reports
- `bench/competitive/adapters/_oracle.mjs` supports text-in-snapshot, url-contains, json-shape-match, selector-state
- `bench:compete --reps N` enforces ≥3 reps unless `--allow-quick-check`
- Stats lib (`scripts/lib/stats.mjs`) has Wilson, bootstrap, Cohen's d, Mann-Whitney, spread verdict
- bad has Gen 7.2 placeholder substitution + auto-complete-from-runScript so extraction tasks work

### What exists but isn't integrated
- Video files exist on disk but are never linked from comparison.md or summary.json
- The bench/scenarios/specs/ folder has rich experiment specs but no real-public-web ones
- `bench/external/webvoyager/` and `bench/external/webarena/` exist but are dormant — they could supply real tasks but the runners are framework-specific and not wired into bench:compete

### What was tested and failed
- Real-web tasks have NEVER been benched. Zero data points.
- The webvoyager harness was never run with the planner

### What doesn't exist yet
- A real-public-web task corpus at all
- A way to handle anti-bot blocks (cloudflare, captcha) gracefully — currently they'd just fail
- Per-task budget caps in the schema (extraction = fast budget, multi-step = slow budget)
- An HTML dashboard that surfaces videos + per-task pass/fail in one view
- Automated retries / "blocked" classification for unreachable sites (separate from genuine fails)

### Measurement gaps

## Current Baselines

- v0.21.0 dashboard-extract: 100% pass (5/5), 7.7s mean, $0.013

## Diagnosis

Architectural maturity is high (Gen 7.2 closed the last known classification of failures). The bottleneck isn't agent code — it's **eval honesty**. We have no evidence `bad` works outside our own fixtures. Anyone reading the README should see real evidence, not a synthetic comparison. The Gen 8 fix is to build the gauntlet, run it, and ship the artifacts.

## Generation 8 Design

### Thesis
Build a 10-task **real-public-web gauntlet** with video evidence, deterministic oracles, multi-rep validation, anti-bot classification, and a public-friendly HTML dashboard. Run it. Show the truth.

### 10 candidate tasks (AWAITING USER APPROVAL)

These are real public sites. No auth required. Each has a deterministic external oracle. I picked them to span the failure modes we care about: extraction, multi-step navigation, dynamic SPAs, paginated lists, search-then-extract, structured data, long documents.

| # | task id | site | category | goal (short) | oracle |
|---|---|---|---|---|---|
| 1 | `hn-top-story-score` | news.ycombinator.com | extraction | Open Hacker News, return JSON `{title, score, comments}` for the #1 story | `json-shape-match` — title is a non-empty string, score is a number 1-9999, comments matches `re:^\d+$` |
| 2 | `wikipedia-fact-lookup` | en.wikipedia.org | search + extract | Search for "Ada Lovelace", return her birth year as JSON `{year}` | `json-shape-match` — `year: "re:^1815$"` |
| 3 | `github-pr-count` | github.com | extraction (real product UI) | Open `github.com/microsoft/typescript/pulls`, return JSON `{open_count}` matching the visible badge | `json-shape-match` — `open_count: "re:^\\d+$"` and the same number appears in the page text |
| 4 | `mdn-array-flatmap` | developer.mozilla.org | technical content extraction | Search MDN for `Array.prototype.flatMap`, return JSON `{returnType, signature}` | `json-shape-match` — `returnType: "re:Array"`, `signature: "re:flatMap"` |
| 5 | `npm-package-downloads` | npmjs.com | lookup + extraction | Open the npm page for `react`, return JSON `{weekly_downloads}` matching the displayed weekly download count | `json-shape-match` — `weekly_downloads: "re:^[\\d,]+$"` |
| 6 | `arxiv-paper-abstract` | arxiv.org | academic search + extract | Find arXiv paper `2310.06825` ("Mistral 7B"), return JSON `{title, first_author, abstract_first_50_chars}` | `json-shape-match` — `title: "re:Mistral"`, `first_author: "re:.+"`, `abstract_first_50_chars` length ≥ 30 |
| 7 | `reddit-subreddit-titles` | old.reddit.com | paginated list extraction | Open `old.reddit.com/r/programming`, return JSON `{titles: [string,string,string]}` of the top 3 post titles | `json-shape-match` — `titles` is an array of 3 non-empty strings (use old.reddit to dodge anti-bot) |
| 8 | `stackoverflow-answer-count` | stackoverflow.com | search + extract | Open the SO question `https://stackoverflow.com/questions/19590641/`, return JSON `{accepted_answer_score, answer_count}` | `json-shape-match` — both fields are integers ≥ 1 |
| 9 | `w3c-html-spec-find-element` | html.spec.whatwg.org | long-document navigation | Find the `<dialog>` element definition in the WHATWG HTML spec, return JSON `{categories: string}` (the "Categories" line for `<dialog>`) | `json-shape-match` — `categories: "re:Flow content"` |
| 10 | `python-docs-method-signature` | docs.python.org/3 | search-in-doc | Look up `dict.update` in the Python 3 docs, return JSON `{signature}` matching the function signature line | `json-shape-match` — `signature: "re:update\\("` |

**Why these 10 (not other 10):**
- All 10 are reachable without auth
- All 10 have deterministic ground truth that doesn't change daily (Mistral 7B abstract, Ada Lovelace's birth year, Python `dict.update` signature, etc.)
- 5 are pure extraction, 3 are search-then-extract, 2 are multi-step navigation
- They span 9 distinct domains so anti-bot blocks on any one site won't kill the whole gauntlet
- They include the failure modes that matter: long pages (W3C spec), SPA loading (npmjs), paginated lists (reddit), structured data behind clicks (github PRs), code-block extraction (Python docs)
- I deliberately avoided sites known to be cloudflare-aggressive (cnn, wsj, twitter) and sites where ground truth changes minute-to-minute (live news, stock prices)

**Approval format:** reply with "approve all" / "approve except [N, N]" / "swap N for X" / "use these 10 instead: [list]". I will not run anything until you confirm.

### Changes (ordered by impact)

#### Architectural (must ship together)
1. **Real-task corpus** under `bench/competitive/tasks/real-web/` — 10 task JSON files with the schema we already have, plus 1 new optional field: `expectedDurationSec` for budget tuning per task type
2. **Anti-bot detection layer** in the bad adapter — if the run hits `chrome-error://`, "Just a moment", "Verifying you are human", or any cloudflare/recaptcha marker in the snapshot, mark `success: null, blocked: true, blockReason: ...` instead of failing. Bench reports it as "blocked" not "failed" so we don't conflate site-side blocks with agent failures
3. **Video surfacing in bench:compete** — update the comparison.md generator to include a relative-path link to `recording.webm` for every cell, AND add a new `dashboard.html` that embeds the videos with `<video>` tags so the user can watch them inline
4. **Per-task budget caps** — task schema gains `expectedDurationSec` and `expectedTokenBudget`; the runner enforces a soft cap and reports overage as a separate metric

#### Prompt/Config (independent, can test separately)
5. **Real-web planner config** at `bench/scenarios/configs/planner-on-realweb.mjs` — same as `planner-on.mjs` plus `supervisor.maxInterventions: 4` (real sites need more recovery)

#### Measurement (eval changes)
6. **Gauntlet-summary.json** — top-level rollup that aggregates all 10 tasks, computes per-framework: pass rate, blocked rate, mean wall-time, mean cost, p95 wall-time, total tokens. This is the headline number.
7. **HTML dashboard** at `<out>/dashboard.html` — embeds all 10 videos, per-task pass/fail, expandable per-rep details, a "share this" copy button. This is what we'd link from the README.

#### Infrastructure (reliability, observability)
8. **Storage state for github** if needed (probably not — public PRs page works without auth)
9. **Network resilience** — retry network errors once before declaring fail (the agent already does this internally; bench harness needs to NOT count first-try failures as architectural failures)
10. **Run bundle archival** — the gauntlet output gets a stable directory name (`gauntlet-YYYY-MM-DD-vX.X.X`) so we can diff future gauntlets against this one

### Alternatives Considered

- **Use webvoyager directly.** Rejected because webvoyager tasks are framework-specific and our existing webvoyager runner doesn't go through bench:compete. Would need to rebuild the integration anyway. Better to start with our own 10 and add webvoyager as Gen 8.5.
- **Run against random websites picked at runtime.** Rejected because the user can't pre-approve, oracles can't be deterministic, and reproduction is impossible.
- **Run the same 10 tasks on browser-use too.** Will do this if you approve — but only after bad alone passes the 10. No point in head-to-head if our own gate fails first.
- **Use vision/screenshot oracles.** Rejected for v1 — text-in-snapshot + json-shape-match are deterministic and don't need a vision LLM judge. Vision oracles can come in Gen 8.5.

### Risk Assessment

- **Anti-bot blocks** — I expect 1-2 of the 10 tasks to be intermittently blocked (cloudflare on github sometimes, npm sometimes). Mitigation: anti-bot detection layer marks them as `blocked` not `failed`. Failure mode is honest reporting.
- **Site UI changes** — real sites change. The oracles use loose regex (e.g. `re:^\d+$` for numbers) so cosmetic changes don't break them. Hard ground truths (Ada Lovelace's birth year, Mistral 7B title) are immutable.
- **Cost** — 10 tasks × 3 reps × ~$0.05 average = ~$1.50 per gauntlet run. 5-rep validation = ~$2.50. Affordable.
- **Wall time** — 10 tasks × 3 reps × ~30s average = 15 minutes. Acceptable.
- **gpt-5.2 quality on real DOM** — biggest unknown. The fixture sites have clean ARIA snapshots; real sites have ad iframes, tracking pixels, and 5x bigger DOMs that hit the snapshot budget cap. Mitigation: if the snapshot budget is the bottleneck, gen 8.5 raises the cap or adds DOM filtering.
- **Reversibility** — all changes are additive (new task files, new reporter helpers, new bench config). Existing benches keep working unchanged. Rollback is `git revert <pr-sha>`.

### Success Criteria

- **All 10 tasks have a working oracle that produces a clear pass/fail when given a known-good result text** (validated by hand-feeding the expected JSON before running the agent)
- **bad's gauntlet pass rate ≥ 70% on first 3-rep run** (I'd target 90% but real-web is hard; 70% is the bar for "this generation worked")
- **At least 1 task at each of the 3 categories (extraction / search+extract / nav) passes** (proves the mechanism works across task types)
- **Per-task video plays inline in the HTML dashboard** (proves the artifact pipeline works)
- **The gauntlet is reproducible** — running it twice in a row produces stable pass-rate ± 1 task variance
- **The dashboard.html is shareable** — paste the HTML in a browser and it looks right without a server
